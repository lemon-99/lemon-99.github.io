String

  • 头文件 #include [注意：与#include <string.h> 的区别]

    using namespace std;

  • string 的定义

    string str; //类似于基本数据类型的定义

    string str = “abcd”; // 可以在声明时对其进行初始化

  • 读入和输出整个字符串，使用cin和cout

    使用printf来输出需要借助c_str()将string类型转化为字符数组进行输出，printf(“%s\n”,str.c_str());

  • string 的内容访问

    • 下标访问

      类似字符数组方式访问

    • 迭代器访问

      • insert()函数和erase()函数要求以迭代器为参数

      • 定义：string::iterator it;

      • 通过*it访问string每一位

  • string常用函数

    • operator +=

      string 的加法，直接将两个string拼接起来

      输出结果：

    • compare operator

      使用==、！=、<、<=、>、>=来进行大小的比较，比较规则是字典序

    • length()/size()

      该函数返回string的长度，即字符个数，时间复杂度O(1)，【注意：不包含’\0’】

    • insert()

      时间复杂度O(N)

      • insert(pos,string)

        在pos位置插入字符串string

      • insert(it,it2,it3)

        it为原字符串插入位置，it2和it3为待插字符串的首尾迭代器，用来表示串[it2,it3)将被放在it位置上。

    • erase()

      时间复杂度O(N)

      • erase(it)

        通过迭代器it删除单个元素

      • erase(first,last) 【注意：迭代器区间是 [first,last) 】

      • erase(pos,length)

        pos 为需要开始删除的起始位置， length 为删除的字符个数

    • clear()

      • 清空string中的数据，时间复杂度O(1)

    • substr()

      • substr(pos,len) //返回pos位置开始，len为长度的子串，时间复杂度O(len)

    • string::npos

      • string::npos 是一个常数，本身的值是 -1 或是 4294967295【注意：此类型是 unsigned_int 类型，此值是32位机器下的结果，64位机器结果是18446744073709551615】
      • string::npos 用以作为 find() 函数失配时的返回值

    • find()

      str.find(str2)

      • str2 是 str 子串，返回在str中出现的第一次位置
      • str2 不是 str 的子串，返回string::npos

      str.find(str2,pos)

      • 从 str 的 pos 位置开始匹配 str2 ，返回值与上面相同
      • 时间复杂度 O(mn) , m 和 n 表示 str 和 str2 的长度

    • replace()

      str.replace(pos,len,str2)

      • 把 str 从 pos 位置开始，长度len的子串替换为 str2

      str.replace(it1,it2,str2)

      • 把 str 的迭代器 [it1,it2) 区间内的子串替换成 str2

map

  • 翻译：映射 //定义数组 int array[100] 时，从 int 型到 int 型的映射，如：array[0] = 25，就是将0映射到25

​ //也可以定义double array[100]，建立了 int 型到 double 型的映射

​ //只能由 int 型到其他类型的映射，带来不便，如：字典，“字符串–>页码“

  • map 可以将任何基本类型映射到基本类型 【注意：这里包括STL容器】

  • 头文件 #include

    using namespace std;

  • 定义map

    • map<typename1,typename2> mp; //前面是映射前类型（键key），后面是映射后类型（值value）
    • map<string,int> mp; //字符数组是不能做键值的，可以使用string
    • map<set,string> mp;

  • 元素访问

    • 下标访问

    • 迭代器访问

      • 迭代器定义

        map<typename1,typename2>::iterator it;

      • map 可以使用 it->first来访问键，使用 it->second 来访问值

        【注意：这里map会按键大小自动排序，和set一样，内部使用红黑树实现】

  • 常用函数

    • find()
      • find(key) //返回键为key的映射的迭代器，时间复杂度O(lonN)
    • erase()
      • mp.erase(it) //删除迭代器it指向的元素，时间复杂度O(1)
      • mp.erase(key) //key为欲删除的映射的键，时间复杂度O(logN)
      • mp.erase(first,last) // first 和 last 均为迭代器，区间注意左开右闭
    • size()
      • 获取map中映射对数，时间复杂度O(1)
    • clear()
      • 清空map中的所有元素，时间复杂度O(N)

  • 常见用途

    • 需要建立字符（或字符串）与整数之间映射的问题，使用map可以减少代码量
    • 判断大整数或者其他类型数据是否存在的问题，可以把map当bool数组使用
    • 字符串和字符串的映射问题

【注意】map中键和值是唯一的，如果一个键对应多个值，只能使用multimap。C++ 11标准增加unordered_map，以散列替代map内部红黑树实现，实现只映射不排序的需求，速度比map要快得多

queue

  • 头文件 #include

    ​ using namespace std;

  • 定义

  • 元素访问

    • 只能通过 front() 和 back() 来访问队首元素和队尾元素

  • 常用函数
    • push()
      • push(x) //将x进行入队，时间复杂度O(1),
    • front() / back()
      • 分别获得队首和队尾元素，时间复杂度O(1)
    • pop()
      • 令队首元素出队，时间复杂度O(1)
    • empty()
      • 检测queue是否为空，返回 true 则空，返回 false 则不空，时间复杂度O(1)
    • size()
      • 返回queue内元素的个数，时间复杂度O(1)
  • 常见用途
    • 广度优先搜索，可以使用此队列
    • 注意使用 front() 、pop() 函数时，必须使用 empty() 判断队列是否为空
    • STL容器中还有两种容器跟队列相关，分别是双端队列 ( deque ) 和优先队列 ( priority_queue ) ，前者是首尾都可以插入删除的队列，后者是使用堆实现的默认将当前队列最大元素置于队首的容器

priority_queue

  • 称为优先队列，底层是使用堆来实现的，队首元素一定是当前队列中优先级最高的那个，

    • emmm，大根堆与小根堆

  • priority_queue定义

    • #include
    • using namespace std;
    • priority_queue name;

  • 元素访问

  • 优先队列无 front() 函数和 back() 函数，只能通过 top() 函数来访问队首元素（堆顶元素），也是优先级最高的元素

  • 常用函数

    • push()
      • push(x) 令x入队，时间复杂度 O(logN)
    • top()
      • 可获得队首元素（即堆顶元素），时间复杂度O(1)
    • pop()
      • 队首元素出队，时间复杂度 O(logN)
    • empty()
      • 检测优先队列是否为空，返回 true 则为空，时间复杂度度O(1)
    • size()
      • 返回优先队列内元素个数，时间复杂度O(1)

  • priority_queue 内置元素优先级的设置

    • 基本数据类型的优先级设置

      • 优先队列对他们的设置一般是数字大的优先级越高
